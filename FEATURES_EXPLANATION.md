FEATURES EXPLANATION — Privacy Policy, Cookie Policy, Cookie Banner, Form Integrations
========================================================================================


1. ZOHO API URL — "https://www.zohoapis.eu/crm/v2 выдаёт Недопустимый URL-адрес"
----------------------------------------------------------------------------------

ПРИЧИНА: Это нормальное поведение. Если открыть URL https://www.zohoapis.eu/crm/v2
напрямую в браузере (или вызвать GET на корневой путь), Zoho вернёт ошибку — потому
что /crm/v2 само по себе не является конечной точкой. Конечные точки — это:
  /crm/v2/Leads, /crm/v2/Leads/search, /crm/v2/Contacts и т.д.

Это НЕ значает, что URL неверный. URL правильный. Ошибка возникает только при прямом
обращении к корню без авторизации и без указания конкретного ресурса.

Серверный код обращается к /crm/v2/Leads и /crm/v2/Leads/search — эти запросы
выполняются корректно.


2. КАК РАБОТАЕТ COOKIE БАННЕР
-------------------------------

При первом посещении сайта внизу экрана появляется баннер:
  "We use cookies. We use cookies to improve your experience... Learn more in our Cookie Policy."

Кнопки:
  - "Accept" — сохраняет в localStorage: { accepted: true }
  - "Decline" — сохраняет в localStorage: { accepted: false }

После нажатия любой кнопки баннер скрывается и больше не показывается
(localStorage key = "cookie_consent").

ЧТО ЭТО ДАЁТ: Это юридически обязательное информирование пользователя о сборе cookies
(требование GDPR). Технически баннер сам по себе не включает/выключает Google Analytics
или другие трекеры — для полноценного управления consent нужно было бы интегрировать
Consent Mode v2 от Google. На данный момент баннер выполняет информационную функцию и
фиксирует выбор пользователя в localStorage.

СВЯЗЬ С MAILERLITE/ZOHO CRM: Баннер НЕ имеет прямой связи с MailerLite или Zoho CRM.
Их данные собираются только через контактную форму.


3. МАРКЕТИНГОВЫЙ ЧЕКБОКС — ГДЕ НАХОДИТСЯ И ЗАЧЕМ
---------------------------------------------------

Чекбокс "I want to receive occasional updates and marketing emails." находится
ТОЛЬКО в форме контактов ("Let's talk about your project" на странице /contact).

В баннере кукис этого чекбокса НЕТ (был удалён — он там не нужен и не соответствует
логике задачи). В Privacy_Policy_and_Coockies.txt описано, что чекбокс маркетинговой
рассылки должен быть под формой — именно там он и находится.

КАК РАБОТАЕТ:
  - Если пользователь отмечает чекбокс и отправляет форму → он добавляется в MailerLite
    как подписчик с группой (если MAILERLITE_GROUP_ID задан в .env)
  - Если чекбокс НЕ отмечен → в MailerLite НЕ добавляется, но в Zoho CRM лид всё равно
    создаётся


4. БАННЕР КУКИС — ИСПРАВЛЕННОЕ ПОВЕДЕНИЕ
-----------------------------------------

После исправления баннер:
  - Показывается при первом посещении (localStorage пуст)
  - Accept/Decline записывают выбор в localStorage
  - Больше не содержит чекбокс маркетинга (убран)
  - Ссылка "Cookie Policy" ведёт на страницу /cookie-policy


5. ОШИБКА ФОРМЫ "Please complete all required fields"
-------------------------------------------------------

ПРИЧИНА БАГА: Текст ошибки был привязан к переменной isSubmitDisabled, которая
становится true как когда есть ошибки валидации (hasValidationErrors = true), ТАК И
когда форма отправляется (isSubmitting = true). Поэтому в момент нажатия Submit и
начала отправки (даже если все поля верны) ошибка кратко появлялась.

ИСПРАВЛЕНО: Текст ошибки теперь привязан только к hasValidationErrors. Он показывается
только когда реально есть ошибки в заполнении полей, и не показывается во время отправки.


6. MAILERLITE — ТОЛЬКО EMAIL, ИМЯ, ФАМИЛИЯ
--------------------------------------------

ЭТО НОРМАЛЬНО. MailerLite — это система для email-рассылок. В ней хранятся:
  - Email (обязательно)
  - Имя (fields.name)
  - Фамилия (fields.last_name)
  - Группа подписчиков (если MAILERLITE_GROUP_ID задан)

Остальные данные (компания, телефон, бюджет, описание проекта) хранятся в Zoho CRM
как атрибуты Лида — MailerLite для этого не предназначен.

Интеграция работает корректно: если пользователь отметил чекбокс "I want to receive
occasional updates and marketing emails." и отправил форму — он появится в MailerLite
как активный подписчик.


7. ZOHO CRM — ЛИДЫ НЕ СОЗДАВАЛИСЬ (ИСПРАВЛЕНО)
------------------------------------------------

ПРИЧИНА БАГА: В коде поиска лида по email:

  const searchData = await searchRes.json()  // ← здесь была ошибка

Когда лид с таким email НЕ существует, Zoho возвращает HTTP 204 (No Content) — пустой
ответ без тела. Вызов .json() на пустом ответе выбрасывал исключение. Это исключение
перехватывалось внешним try/catch, который добавлял "crm" в массив errors и пропускал
создание лида.

ИТОГ: При КАЖДОЙ отправке формы с новым email лид в Zoho НЕ создавался.
(Если бы email уже существовал — Zoho вернул бы 200 с данными, JSON парсился бы
успешно и лид тоже не создавался, но по правильной причине — дубликат.)

ИСПРАВЛЕНИЕ: Добавлена явная проверка на статус 204:

  if (res.status === 204) {
    return false; // лид не найден → создавать
  }

Теперь:
  - 204 → лид не существует → создаём
  - 200 с данными → лид уже есть → не создаём
  - Ошибки API → логируются в консоль сервера → лид не создаётся

Дополнительно добавлено поле Last_Name || First_Name как запасной вариант, если
фамилия случайно не передана (Last_Name обязателен в Zoho CRM API).

КАК ПРОВЕРИТЬ РАБОТУ ПОСЛЕ ИСПРАВЛЕНИЯ:
  1. Запустить сервер (npm run dev)
  2. Открыть /contact, заполнить форму (все поля)
  3. Нажать "Submit enquiry"
  4. Зайти в Zoho CRM → Leads → должен появиться новый лид
  5. Если лид не появился — проверить консоль Next.js сервера на наличие ошибок
     вида "[Zoho] ..."


8. КАК ДОБАВИТЬ ЖИРНЫЙ ТЕКСТ В СТРАНИЦЫ ПОЛИТИК
-------------------------------------------------

Страницы Privacy Policy (/privacy-policy) и Cookie Policy (/cookie-policy) теперь
поддерживают HTML-разметку в тексте. Текст берётся из:

  crocode-shopify/public/locales/en.json

Чтобы сделать слово или фразу жирной:
  Используйте тег <strong> непосредственно в JSON-строке.

ПРИМЕР — В en.json:

  "section1": {
    "title": "1. Data Controller",
    "text": "The controller of your personal data is:\n<strong>KROKOD sp. z o.o.</strong>\nul. Tadeusza Rejtana..."
  }

ПРИМЕР — Для элементов списка:

  "items": [
    "First name, <strong>last name</strong>",
    "Company name",
    "Email address"
  ]

Поддерживаемые теги: <strong>, <em>, <br>, <a href="...">, <u>

ВАЖНО: Не используйте теги <script>, <img>, <style> — они нежелательны в данном
контексте. Только форматирующие теги.

Аналогичный механизм используется в HomeAboutUs (секция "message" в HomePage.why-shopify.message
через компонент RichText).


9. СТРУКТУРА ИНТЕГРАЦИЙ (КРАТКАЯ СХЕМА)
----------------------------------------

Пользователь заполняет форму → нажимает Submit enquiry
  ↓
POST /api/submit-form
  ├── Если newsletter = true → MailerLite: добавить подписчика
  │     (email + firstName + lastName + groupId)
  └── Всегда → Zoho CRM:
        1. GET /crm/v2/Leads/search → проверить, есть ли лид с таким email
           - 204 (не найден) → создать лид
           - 200 с данными → лид уже есть, не создавать
        2. POST /crm/v2/Leads → создать лид
              (First_Name, Last_Name, Company, Email, Phone, Lead_Source, Description)
